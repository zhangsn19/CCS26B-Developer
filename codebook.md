# Codebook of The Interviews

The following presents the codebook for the qualitative interviews.

#### RQ1: Understanding, Responsibility, and Knowledge Sources of AI Agents' Risks

| Themes | Codes |
| :--- | :--- |
| Understanding of AI Agents' Risks and Developers' Responsibilities | - High salience of user-facing safety risks <br> - Limited awareness of systemic security vulnerabilities <br> - Externalization and underestimation of privacy risks <br> - Externalization of responsibilities for AI agents' risks | 
| Knowledge Sources Informing Developers' Understanding | - Media and online communities <br> - Academic and industry sources <br> - Interpersonal <br> - Personal experience |

#### RQ2: Security and Privacy Practices

| Themes | Codes |
| :--- | :--- |
| Designing Stage: Proactive Risk Mitigation via Data Minimization and Platform Selection | - Proactive data handling policies <br> - Strategic platform and database selection |
| Building Stage: Implementation via Ad-Hoc Methods and Manual Safeguards | - Implementation of technical safeguards <br> - Ad-hoc tooling and manual implementation |
| Testing Stage: Informal Communication and Reactive Delineation of Responsibility | - Informal mechanisms for user communication <br> - Reactive delineation of responsibility |

#### RQ3: Inhibitors When Implementing Security and Privacy Practices

| Themes | Codes |
| :--- | :--- |
| Motivational Inhibitors | - Indifference about privacy <br> - Functional priorities over security <br> - Unclear division of responsibility |
| Regulational Inhibitors | - Deficiencies in legal and regulatory frameworks |
| Resources Inhibitors | - Individual resource constraints <br> - Lack of ecosystem support |
